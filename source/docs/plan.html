FogCloud Plan

Step 1: Secure File Sync

The basic problem with cloud services is having your personal data stored on untrusted computers. The first component of the FogCloud system will be a secure file sync tool. This is useful itself as a replacement for tools like Bittorrent Sync and Dropbox.

Further, it provides the basic common functionality needed to implement secure versions of all the other cloud services with shared local data.

Step 2: Address Book

Your address book can be stored as a directory of vCard or similar and then synced with the file sync tool.

Step 3: Email and IM

There are two problems here: Email delivery and syncing archived email. The second problem is solved trivially by storing it locally in files and then syncing it.

Delivery is slightly more complicated. There are two cases:

  1. Secure email should be delivered without revealing it. This requires a FogCloud-to-FogCloud message protocol.
  2. For insecure email, the best we can do is encrypt on receipt.

IM isn't different from email for our purposes.

This requires either new email apps for desktop and mobile, or the extension of existing programs to support our format for contacts and archive storage.

Step 4: Calendar, Browser Data

These can be stored as files and synced. A standard calendar file format is iCalendar.

Step 5: Text Messages

This requires a phone app that reads text messages. They should be stored as IMs.
